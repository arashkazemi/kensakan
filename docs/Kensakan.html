the javascript code to compile/run.
step_callback function null

the callback handler for stepping, is called when runner enters a breakpoint or steps to next line. If the step callback returns true, the execution will be paused until one of the functions step() or continue() is called. If it returns false the ececution continues to the step or breakpoint.
stop_callback function null

is called when the function running is finished.
step_loop_args boolean true

if set, it will also step into loop test and update expressions, see Kensakan#step_loop_args

Members

breakpoints

Set the breakpoint lines in this object to anything other than false or null, before calling prepare(), like:

    var k = new Kensakan (
           "console.log(1);\nconsole.log(2);\nconsole.log(3);\n"
         );

    k.breakpoints[2] = true;

    k.debug(true); // runs until reaching the breakpoint on line 2 
                   // (output:  1) 

    k.continue() // runs until end as there is no other breakpoints 
                 // (output:  2,3)

step_loop_args

If set, Kensakan will also step into loop test and update expressions, like i<n and i++ in the loop:

       for( let i=0; i<n; i++ ) {...}

You can set step_loop_args, but should call prepare after so it can be applied.
